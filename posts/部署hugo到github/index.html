部署 Hugo 到 GitHub Pages

将 Hugo 生成的网站部署到 GitHub Pages 上是一个常见的做法。以下是一步步的部署指南。

第一步：安装 Hugo

首先确保你已经在本地安装了 Hugo。如果还没有安装，可以根据 Hugo 官方文档进行安装。

第二步：初始化 Git 仓库

进入你的 Hugo 网站目录并初始化 Git 仓库：

cd /path/to/your/hugo/site
git init

创建一个 .gitignore 文件，并添加 Hugo 特有的忽略规则：

echo "/public/" >> .gitignore
echo "/resources/" >> .gitignore

第三步：创建 GitHub 仓库

  1. 登录到 GitHub。
  2. 创建一个新的仓库。例如，你可以命名为 my-hugo-site。
  3. 复制仓库的 URL，例如 https://github.com/yourusername/my-hugo-site.git。

第四步：配置 Hugo 网站生成路径

修改 Hugo 配置文件（通常是 config.toml），添加或修改 baseURL，使其指向你 GitHub Pages 的 URL。例如：

baseURL = "https://yourusername.github.io/my-hugo-site/"

第五步：构建 Hugo 网站

在本地构建 Hugo 网站：

hugo

这会在 public/ 文件夹中生成静态文件。

第六步：添加 GitHub 远程仓库

将你的本地仓库连接到 GitHub 远程仓库：

git remote add origin https://github.com/yourusername/my-hugo-site.git

第七步：配置部署到 GitHub Pages

使用 gh-pages 分支

  1. 创建一个新的分支 gh-pages 用于存放静态文件：

    git checkout -b gh-pages

  2. 将 public/ 文件夹的内容添加到 Git 仓库：

    cp -r public/* .
git add .
git commit -m "Deploy to GitHub Pages"

  3. 推送到 GitHub 上的 gh-pages 分支：

    git push -u origin gh-pages

配置 GitHub Pages

  1. 登录 GitHub，打开你的仓库页面。
  2. 点击 Settings。
  3. 在左侧选择 Pages。
  4. 在 Source 选择 gh-pages 分支，然后保存。

第八步：自动化部署（可选）

为了实现每次更新自动部署，你可以使用 GitHub Actions 来自动化部署过程。以下是一个简单的 GitHub Actions 配置，自动构建和部署 Hugo 网站：

在仓库根目录创建 .github/workflows/deploy.yml 文件，内容如下：

name: Deploy Hugo to GitHub Pages

on:
  push:
    branches:
      - main  # 只在 main 分支更新时触发

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Setup Hugo
      uses: peaceiris/actions-hugo@v2
      with:
        hugo-version: '0.81.0'  # 使用你需要的 Hugo 版本

    - name: Build Hugo
      run: hugo --minify

    - name: Deploy to GitHub Pages
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./public

这个配置会在每次向 main 分支推送时自动构建并将 public 目录中的内容部署到 gh-pages 分支。

第九步：访问你的 GitHub Pages 网站

部署完成后，你可以访问 https://yourusername.github.io/my-hugo-site/ 来查看你的网站。
